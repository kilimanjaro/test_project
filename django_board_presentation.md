# Django 게시판 개발 경험

## 슬라이드 1: 표지 / 프로젝트 개요

**핵심 메시지:** Python과 Django로 게시판을 처음부터 완성까지 직접 구현했다.

**내용:**
- 목적: 게시글 작성·수정·삭제·검색·댓글 기능을 갖춘 웹 게시판
- 기술 스택: Python 3.14 / Django 6.x / SQLite / 순수 CSS
- 개발 방식: TDD(테스트 주도 개발) 적용, GitHub CI/CD 연동

**시각자료 지시사항:** 중앙에 프로젝트 디렉토리 구조 트리 이미지 배치.
오른쪽 상단에 Django 로고 아이콘.

---

## 슬라이드 2: 핵심 구현 내용

**핵심 메시지:** TDD로 품질을 확보하고, Q 객체와 Paginator로 검색·페이지네이션을 구현했다.

**내용:**
- 게시글/댓글 CRUD — `Post`, `Comment` 모델, 함수형 뷰 일관 적용
- 검색 — `Q(title__icontains=q) | Q(content__icontains=q)` OR 검색
- TDD — 테스트 파일 6개, 15개 테스트 케이스 전체 통과

**시각자료 지시사항:** 왼쪽에 TDD 사이클 다이어그램(작성→실패→구현→통과),
오른쪽에 브라우저 게시판 목록 스크린샷.

---

## 슬라이드 3: 결론 및 배운 점

**핵심 메시지:** 직접 만들어보며 Django의 강점과 TDD의 효과를 체감했다.

**내용:**
- Django ORM·마이그레이션으로 SQL 없이 DB 설계 가능
- TDD 덕분에 기능 추가 시 기존 코드 회귀 버그 사전 차단
- GitHub Actions CI로 push마다 자동 테스트 실행

**시각자료 지시사항:** 중앙에 GitHub Actions 통과 화면 스크린샷.
하단에 "Q&A" 텍스트 크게 배치.
